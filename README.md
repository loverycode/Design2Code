# Design2Code

Плагин для Figma, который автоматически проверяет структуру дизайн-макета по заданным правилам и генерирует чистый код (React, HTML, CSS). Плагин обучает дизайнера правильной организации слоёв, а разработчик получает готовый компонент, готовый к интеграции.

## Возможности

-  **Проверка структуры макета**  
  - Auto Layout у всех контейнеров с дочерними элементами  
  - Именование компонентов в `PascalCase` (например `ButtonPrimary`)  
  - Именование слоёв в `kebab-case` (например `hero-section`)  
  - Контроль глубины вложенности (не более 5 уровней)  
  - Динамические текстовые поля: `{{propName}}` → React props  

-  **Генерация кода**  
  - React-компонент (TypeScript / JSX)  
  - HTML-разметка  
  - CSS-стили (конвертация Auto Layout → Flexbox)  

-  **Веб-интерфейс**  
  - Дашборд со списком проектов и компонентов  
  - Просмотр кода с подсветкой синтаксиса (Monaco Editor)  
  - Копирование кода в буфер обмена  
  - Скачивание ZIP-архива (Component.jsx, index.html, styles.css, README.md)  

-  **Обучающая документация**  
  - Встроенные подсказки в плагине  
  - Инструкция по эксплуатации с перечнем ошибок и их решений  

## Требования к окружению

- Node.js (версия 18 или выше)
- npm или yarn
- Docker (для запуска PostgreSQL)
- Figma Desktop App (для использования плагина)

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/loverycode/Design2Code.git
cd Design2Code
```

### 2. Настройка базы данных (PostgreSQL)

Запустите контейнер Docker:

```bash
docker run --name design2code-postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=design2code_db \
  -p 5433:5432 -d postgres:15
```

### 3. Настройка переменных окружения (бэкенд)

Создайте файл `.env` в корневой папке проекта:

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5433/design2code_db?schema=public"
PORT=3001
```

### 4. Установка зависимостей и запуск бэкенда

```bash
npm install
npx prisma migrate dev --name init
npx prisma generate
npm run dev
```

Сервер запустится на `http://localhost:3001`

### 5. Запуск фронтенда

```bash
cd frontend
npm install
npm start
```

Веб-интерфейс станет доступен на `http://localhost:3000`

### 6. Установка плагина в Figma

1. Откройте Figma Desktop App  
2. Перейдите в меню: **Plugins → Development → Import plugin from manifest…**  
3. Выберите файл `manifest.json` из папки проекта  
4. Плагин появится в списке **Plugins → Design2Code**

## Использование

### Для дизайнера (в Figma)

1. Подготовьте макет согласно правилам (см. подсказки в плагине или ниже)  
2. Выделите корневой фрейм  
3. Запустите плагин: **Plugins → Design2Code**  
4. Нажмите **«Проверить структуру»**  
5. Исправьте все обнаруженные ошибки  
6. Нажмите **«Экспортировать»**  
7. Скопируйте полученную ссылку и отправьте разработчику  

**Правила подготовки макетов:**  
- У всех фреймов с >1 дочерним элементом включён **Auto Layout** (`Shift+A`)  
- Компоненты называйте в `PascalCase` (например `UserCard`)  
- Обычные слои – в `kebab-case` (например `footer-links`)  
- Глубина вложенности – не более 5 уровней  
- Динамический текст оборачивайте в `{{ }}` (например `{{buttonText}}`)

### Для разработчика (веб-интерфейс)

- Перейдите по полученной ссылке  
- Выберите нужный формат кода: React, HTML или CSS  
- Скопируйте код в буфер или скачайте ZIP-архив  
- В архиве содержатся файлы: `Component.jsx`, `index.html`, `styles.css`, `README.md`
